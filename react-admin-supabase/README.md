## React-Admin + Supabase (Vite + TypeScript)

This project scaffolds a React-Admin dashboard backed by Supabase.

### Prerequisites
- Node 18+
- A Supabase project with a `posts` table (id uuid or int primary key, plus fields you need)

### Environment Variables
Create a `.env` file in the project root with:

```
VITE_SUPABASE_URL=https://YOUR-PROJECT.ref.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
```

### Run Locally
```
npm install
npm run dev
```

Open the dev server URL printed in the terminal.

### Build
```
npm run build
npm run preview
```

### Where Things Live
- `src/supabaseClient.ts`: Supabase client initialization
- `src/dataProvider/supabaseDataProvider.ts`: React-Admin data provider mapping CRUD to Supabase
- `src/authProvider/supabaseAuthProvider.ts`: Email/password auth via Supabase
- `src/App.tsx`: React-Admin `Admin` using a `posts` resource via guessers

### Notes
- The data provider assumes a primary key column named `id` on your tables.
- For RLS, configure policies in Supabase to allow the required read/write ops.

# React + TypeScript + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Babel](https://babeljs.io/) (or [oxc](https://oxc.rs) when used in [rolldown-vite](https://vite.dev/guide/rolldown)) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).

])
```

        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```
